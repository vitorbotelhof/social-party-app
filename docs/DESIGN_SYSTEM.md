# ENTRE NÓS — DESIGN SYSTEM

## Design Philosophy

The product should feel:

* premium
* social
* cinematic
* emotional
* modern
* intimate
* human
* nightlife-oriented

The app is NOT:

* gamer UI
* childish UI
* meme UI
* hyper-casual UI
* generic startup UI

The visual language should communicate:
“something interesting happens when people gather.”

---

# Visual Identity

Core aesthetic:

* dark mode first
* cinematic lighting
* soft depth
* elegant spacing
* subtle glow
* modern minimalism
* emotional atmosphere

Visual references:

* Spotify
* Netflix
* Discord
* Apple Music
* modern nightlife apps

# Typography

Style:

* modern sans-serif
* rounded
* highly readable
* elegant spacing

References:

* Inter
* Satoshi
* Circular
* Plus Jakarta Sans

---

# Typography Rules

* Avoid heavy bold everywhere
* Prefer visual hierarchy through spacing
* Large breathing room
* Short readable text blocks
* Strong contrast between headings and body

---

# Spacing System

Use spacious layouts.

Spacing feeling:

* breathable
* elegant
* premium
* uncluttered

Prefer:

* large paddings
* generous margins
* visual calm

Avoid:

* cramped layouts
* dense UI
* excessive information

---

# Corner Radius

Use soft rounded corners.

Standard radius:

* cards: 20-24
* buttons: 16-20
* modals: 28-32

Avoid sharp corners.

---

# Depth System

Depth should feel soft and cinematic.

Use:

* subtle shadows
* layered surfaces
* soft gradients
* glow highlights

Avoid:

* hard shadows
* aggressive neumorphism
* excessive blur

---

# Glow System

Glow should be:

* subtle
* elegant
* atmospheric

Primary glow:

* purple soft glow
* low opacity
* soft spread

Glow is used to create:

* nightlife atmosphere
* emotional warmth
* interaction focus

Never use:

* aggressive neon glow
* gamer glow effects

---

# Motion Language

Animations should feel:

* smooth
* soft
* cinematic
* organic

Animation priorities:

* fluid transitions
* emotional pacing
* responsive interactions
* polished microinteractions

Avoid:

* chaotic motion
* hyperactive UI
* exaggerated bounces

---

# Component Philosophy

Components should feel:

* tactile
* soft
* elegant
* lightweight

Prefer:

* clean cards
* layered surfaces
* soft gradients
* subtle transparency

Avoid:

* excessive borders
* noisy visual elements
* visual clutter

---

# Buttons

Buttons should:

* feel premium
* have strong hierarchy
* feel touch-friendly
* use soft depth

Primary buttons:

* purple gradients
* subtle glow
* large tap area

Avoid:

* tiny buttons
* flat boring CTAs
* hyper saturated buttons

---

# Cards

Cards are the foundation of the UI.

Cards should:

* feel elevated
* create focus
* separate emotional contexts

Use:

* soft surfaces
* subtle gradients
* layered depth

---

# Modal Philosophy

Modals should feel:

* immersive
* cinematic
* focused

Use:

* blurred backgrounds
* centered content
* emotional emphasis

---

# Lobby and Multiplayer Feel

Multiplayer spaces should feel:

* alive
* social
* realtime
* emotionally active

Use:

* player presence
* soft activity animations
* reactions
* live indicators

The room should feel like:
“people gathering together.”

---

# Emotional UI Priorities

Optimize visual experience for:

* anticipation
* suspense
* laughter
* social tension
* reveals
* reactions

UI should amplify emotional pacing.

---

# Accessibility Rules

* maintain readable contrast
* avoid tiny touch targets
* prioritize readability in dark mode
* support fast comprehension

---

# Consistency Rules

All screens must feel:

* cohesive
* premium
* cinematic
* emotionally connected

Never allow:

* random styles
* inconsistent spacing
* mixed aesthetics
* generic components

---

# Non-Negotiable Rule

Every screen must feel like:
“a premium social experience happening at night.”
