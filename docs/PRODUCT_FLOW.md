# ENTRE NÓS — PRODUCT FLOW

## Core Philosophy

**emoção → intenção social → logística**

Never: logística → emoção

Every step of the flow should feel like entering an experience, not configuring a system.

---

# Step 1 — Game Selection

The user selects a social experience:

* Mr White
* Eu Nunca
* Most Likely To
* etc.

The game catalog is the primary navigation structure.

Each game card should communicate:

* tensão emocional
* energia social
* o que vai acontecer entre as pessoas
* atmosfera — não categoria

Cards are movie posters, not app thumbnails.
They should make the user feel: "quero viver isso."

---

# Step 2 — Social Dynamic Selection

The user chooses HOW the group will play:

* 1 device (passado entre jogadores)
* each player on their own device
* join existing room

Available options depend on the game.

Games define:

* supportsLocal
* supportsRealtime
* supportsRoomJoin

This step is about social intent, not multiplayer logistics.
Language should reflect that: "Como vocês vão jogar?" — not "Select multiplayer mode."

---

# Step 3 — Session Configuration

The user configures the session:

* player count
* timers
* categories
* spicy mode
* custom rules

This is logistics. Keep it fast and minimal.
Sane defaults everywhere.
Most players should never need to change anything.

---

# Step 4 — Gameplay

Gameplay should begin with minimal friction.

Priority:

* fast first laugh
* emotional pacing
* realtime social interaction

The phone is a social catalyst, not entertainment.
Players should be looking at each other, not at the screen.

---

# UX Non-Negotiable

Correct hierarchy:

**emoção → intenção social → logística**

Never:

**logística → emoção**

The user should always feel they are choosing an experience.
Never feel they are configuring infrastructure.

---

# Onboarding Emotional Moment

Before the home screen, the product should signal what kind of experience this is.

Not a splash screen. Not a tutorial.
A single emotional beat: one phrase, dark background, no ornament.

Something like:
"O que vai acontecer entre vocês não fica aqui."

Two seconds. Then home.

This is onboarding as emotional calibration, not user education.

---

# Multiplayer Entry Points

Room creation and room joining should feel like:

* "criar uma sala" → abrir espaço para o grupo
* "entrar numa sala" → ser convidado para algo

Never feel like:
* multiplayer session initialization
* server room selection
* lobby configuration

The language and visual design of the room must feel like a secret invitation,
not a game lobby.
